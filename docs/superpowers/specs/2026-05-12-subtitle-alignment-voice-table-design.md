# 字幕对齐中心配音轨与脚本文案表格设计

## 目标

本次改造让 M08 字幕对齐中心从“只有脚本文案和空字幕轨”变成可连续使用的字幕工作台：页面加载当前项目后，自动读取脚本文档、字幕轨和最新已生成配音轨；生成字幕时优先绑定该配音轨，生成带时间码的字幕段；脚本文案在页面中以表格呈现，方便用户核对段号、时间、目标和字幕文本。

## 用户体验

- 顶部仍保留“AI 自动对齐”和“保存微调与样式”两个主按钮。
- 页面会显示当前项目、字幕版本状态、来源配音状态和脚本文案状态。
- 左侧增加脚本文案表格，列为：段号、时间、段落目标、字幕文案、时间码状态。
- 顶部摘要不再重复展示脚本文案原文，脚本文案只保留表格视图。
- 字幕段列表继续用于编辑生成后的字幕段。
- 中间字幕校对台只保留 9:16 手机预览画面窗口，当前字幕段以叠字方式显示，支持拖动微调字幕位置。
- 右侧继续承载时间码、样式和版本面板，阻断草稿也允许人工微调时间码和字幕样式。
- 样式面板最小字体为 18，并提供行距、字幕框宽度、位置、颜色和背景色调整。
- 没有配音轨时不阻断用户生成草稿，但提示“先生成配音可获得更可靠时间码”。

## 生成与表格体验补充

- 脚本生成 Prompt 需要明确语言边界：面向观众的标题、钩子、口播、字幕和 CTA 展示文案按目标语言输出；制作说明类字段继续使用中文，避免英文脚本把段落目标、画面建议和留存说明也变成英文。
- 分镜生成 Prompt 禁止输出“延续上句”“同上”“见上文”等占位文案；如果某个 shot 不承载口播或字幕，用空字符串；如果承载，则填写脚本原文。
- Runtime 在保存 AI 分镜 JSON 前清理 voiceover / subtitle 中的延续占位，并优先回填来源脚本段的真实口播与字幕文本。
- 脚本和分镜结构化表格必须保留横向滚动和足够列宽，避免英文长句、拍摄注意和画面说明被压缩成过窄竖排。
- 字幕对齐中心生成源文本必须以脚本文案表格行为准；旧纯文本脚本里的 `S01 0-5s` 类前缀只用于表格段号和时间，不得进入字幕文案。

## 后端设计

`POST /api/subtitles/projects/{project_id}/tracks/generate` 保持原地址和请求结构。`sourceVoiceTrackId` 仍可显式传入；如果前端未传，`SubtitleService` 会通过 `VoiceRepository.list_tracks(project_id)` 选择最新 `status="ready"` 的配音轨作为来源。

字幕段生成规则：

- 生成源文本按脚本文案表格行传入 Runtime；Runtime 再防御清理旧脚本行中的段号和时间前缀。
- 英文脚本行按一行一个字幕段处理，不因为 `?` 这类英文标点拆成额外段落。
- 来源配音轨存在且同序号片段有有效 `startMs/endMs`：沿用该时间码。
- 来源配音轨存在但没有片段时间码：按字幕文本长度生成连续时间码，记录来源配音轨快照。
- 来源配音轨不存在：沿用现有本地规则生成草稿，`alignment.status="draft"`，提示先生成配音轨。

本次不声称完成音频级 forced alignment，不新增 ASR Provider 调用，也不直接解析音频波形做精准切词。

## 前端设计

`subtitle-alignment` store 是唯一数据入口：

- 并行读取脚本文档、字幕轨和配音轨。
- 维护 `scriptRows`，由结构化脚本文档或旧 Markdown 转换而来。
- 维护 `sourceVoiceTrack`，默认取最新 `ready` 配音轨。
- `generate()` 请求体加入 `sourceVoiceTrackId`，没有可用配音轨时不传。
- `sourceText` 优先由 `scriptRows` 的字幕文案拼接生成，并在加载历史脏字幕轨道时按脚本文案表格行修正可编辑草稿。
- `SubtitlePreviewStage` 负责 9:16 手机画幅、字幕拖动与偏移回写；`SubtitleStylePanel` 负责行距和字幕框宽度等样式项。

新增 `SubtitleScriptTable.vue` 只负责展示，不做请求。页面把脚本文案表格放在左列上方，让用户在生成字幕前能检查脚本结构。

## 数据与契约

不新增数据库表，不新增 Runtime 路由。契约变化仅体现在 M08 文档语义：

- 字幕生成会自动使用最新可用配音轨。
- 生成结果的 `sourceVoice` 表示来源配音轨快照。
- `alignment.status` 可继续使用 `draft / pending_alignment / aligned / needs_alignment`。
- `SubtitleStyleDto` 增加 `lineHeight / boxWidth / offsetX / offsetY`，旧样式记录通过默认值兼容。

## 异常与日志

- 后端读取字幕轨、配音轨、解析 JSON 失败时继续 `log.exception(...)` 并返回中文 `HTTPException.detail`。
- 前端所有 Runtime 异常继续走 `toRuntimeErrorShape`，页面显示中文错误，不暴露 traceback。
- 缺少配音轨不是错误，只是降级为草稿生成。

## 测试

- Pytest 覆盖自动选择最新配音轨、沿用来源配音时间码、无配音轨降级，以及旧段号时间前缀清理。
- Vitest 覆盖脚本文案表格行提取、store 生成 payload、历史脏轨道修正、9:16 预览拖动、页面表格呈现和状态文案。
- 构建验证覆盖 `npm --prefix apps/desktop run build`。

## 不做事项

- 不新增字幕 Provider 或 ASR Provider。
- 不做逐字级强制对齐。
- 不做字幕模板资产化迁移。
- 不改 AI 剪辑工作台时间线回写，本次只保留 `SubtitleTrack` 层能力。
