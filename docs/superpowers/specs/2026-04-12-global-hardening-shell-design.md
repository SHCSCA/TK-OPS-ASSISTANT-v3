# 第 0 步全局硬化与壳层设计规格

## 目标

TK-OPS 需要从能运行的工程基线，进入可面向真实用户的桌面工作台基线。本规格只定义全局壳层、首启前置页和创作总览，不扩大后续业务范围。

## 视觉基调

- 关键词：本地 AI 视频创作指挥舱、克制、现代、清晰、低噪声。
- 首屏不做普通后台卡片堆砌，而是突出当前项目、创作链路和下一步动作。
- 允许少量磨砂、柔和渐变和扫描线式背景层次，但不得影响信息扫描。
- 所有文案为中文，必要技术词保留 Runtime、Provider、OpenAI。

## 壳层信息架构

- 顶部：产品标识、当前页面、当前项目、关键状态。
- 左侧：16 页中文导航，按启动与总览、创作前置、创作与媒体、执行与治理、系统与 AI 分组。
- 中央：当前路由内容。
- 右侧：页面上下文、当前项目、运行状态、最近错误、授权摘要。
- 底部：Runtime、配置、授权、最近同步状态。

## 首页信息架构

- 无项目：显示创建项目、最近项目空态、系统健康状态、建议下一步。
- 有项目：显示当前项目、脚本/分镜版本、最近访问时间和进入脚本/分镜的主动作。
- 所有数据来自 ProjectStore、ConfigBusStore、LicenseStore。

## 首启前置页

- 保留 BootstrapGate，不渲染主壳层。
- 加载页展示四步检查：Runtime、配置、授权、初始化。
- 授权页展示机器码、复制、授权码输入、完成授权。
- 初始化页展示目录与 AI 默认项，授权成功后才可保存。

## 授权文案

- 未找到授权私钥。
- 授权公钥未配置，无法校验授权码。
- 授权码格式非法。
- 授权码签名校验失败。
- 授权码与当前机器不匹配。
- 当前授权类型不受支持。

## 验收

- 主界面无乱码。
- 壳层不展示组件 import 或开发路径。
- 首页无假统计数字。
- 授权机 BAT 可输入机器码并输出授权码。
- 最终验证命令全部通过。
