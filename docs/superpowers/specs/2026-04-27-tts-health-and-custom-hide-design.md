# TTS 健康检查与自定义入口隐藏设计说明

## 问题

火山豆包语音是 TTS 专项 Provider，但当前健康检查接口先判断 Provider 是否支持 `text_generation`。TTS Provider 不支持文本生成，因此直接返回“尚未接入文本模型连通性检测”，这与用户正在检查语音模型的上下文不一致。

同时，自定义 Provider 暂时过多，会让当前配置界面复杂度过高。现阶段用户主要需要开箱可用的主流厂商和媒体专项接入，自定义能力可以先在 Runtime 保留兼容，前端隐藏入口。

## 决策

1. `check_provider_health` 改为能力分支：
   - `text_generation`：保留原真实文本探测。
   - `tts`：检查密钥、Base URL、TTS 模型目录，返回语音专项状态。
   - 其他专项：返回专项能力尚未接入通用健康检查的中文提示。
2. 前端 Provider Hub 只显示：
   - 模型平台。
   - 媒体专项。
3. 自定义 Provider 暂时不展示：
   - 不出现在 Provider 下拉。
   - 不出现在模板库。
   - 不显示“新增自定义”按钮。

## 兼容性

后端不删除自定义 Provider ID，避免破坏已有配置、数据库记录和契约测试。隐藏只发生在设置页展示层。

## 状态文案

TTS 就绪文案：

`语音合成 Provider 已配置，模型目录可用于配音生成；真实音色、额度和返回音频请在配音中心执行一次生成确认。`
