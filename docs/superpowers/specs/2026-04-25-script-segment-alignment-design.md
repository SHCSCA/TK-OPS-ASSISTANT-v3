# 2026-04-25 脚本策划工作台与 Markdown 预览设计

## 问题定义

当前脚本链路的真实诉求是：

- AI 产出的是 Markdown 脚本；
- 用户需要先用结构化策划字段约束脚本生成；
- 用户需要在脚本工作面直接看到可读预览，也能切回 Markdown 原文编辑；
- 左侧结构锚点和分镜页需要基于同一份正文做结构导航。

现状存在三处偏差：

1. 外围 `````markdown` 代码围栏会被误显示成首段标题。
2. 策划工作台字段不足，无法稳定约束目标用户、视频目的、风格、拍摄条件和禁止内容。
3. 脚本工作面没有直接承担成熟 Markdown 预览职责。
4. 结构锚点和分镜页如果各自维护解析规则，会继续漂移。

## 设计目标

- 用单一 helper 生成统一的“清洗后 Markdown 正文”。
- 策划工作台提供短视频脚本策划字段模板，并将字段组合为生成/改写提示词。
- 脚本页中间工作面提供 Markdown 预览和原文编辑双模式。
- 预览层使用 `markdown-it` 解析，避免继续维护半套 Markdown 解析器。
- 左侧结构锚点和分镜页脚本段落都消费同一份清洗后的正文。
- 标题、列表、表格等主要结构要保留可读性，但外围代码围栏和纯分隔线不能进入正文。

## 策划字段规则

策划工作台包含以下字段：

- 视频主题：必填，是 AI 生成脚本的核心输入。
- 产品/服务：可填，无产品或服务时写“无”。
- 目标用户：描述目标人群，如美国年轻女性、宝妈、宠物主人、办公室人群等。
- 视频目的：从涨粉、引流、带货、种草、品牌曝光、评论互动、私信转化中选择。
- 视频时长：从 15 秒、30 秒、45 秒、60 秒中选择。
- 账号定位：从测评号、剧情号、种草号、专业科普号、工厂号、品牌号、个人 IP 号中选择。
- 视频风格：从真实口播、剧情反转、测评对比、痛点放大、情绪共鸣、搞笑夸张、高级质感、街头采访中选择。
- 拍摄条件：从真人出镜、不露脸、产品实拍、手部演示、图文混剪、AI 视频、工厂实拍中选择。
- 语言要求：从中文、英文、中英双语中选择。
- 禁止内容：可填，用于约束不能出现的词、违规点和夸大承诺。

生成和改写时，前端将字段组合为一段 Markdown 风格的策划摘要，继续通过现有 Runtime 脚本接口提交，不新增后端契约。

## 正文清洗规则

### 1. 预处理

- 统一换行符。
- 去掉首尾空白。
- 若正文被单层 `````markdown` / ````` 包裹，则剥离外围代码围栏。
- 将仅由 `---` / `***` / `___` 构成的行视为分隔线，不产出独立正文段。

### 2. 结构锚点规则

- 识别 Markdown 标题行 `#` 到 `######`。
- 标题出现时开启新段。
- 标题文本去掉 Markdown 前缀，作为锚点标题。
- 若无标题，则退化为按空段切分，但仍忽略纯分隔线。

### 3. 预览规则

- 工作面默认以预览形式展示标题、段落、列表、表格、分隔线等主要结构。
- 预览由 `markdown-it` 渲染，配置 `html: false`，防止用户或 AI 结果中的原始 HTML 直接进入页面。
- 预览只消费清洗后的正文，不直接展示外围代码围栏。
- 原文模式使用现有输入组件编辑完整 Markdown 字符串，保存链路不改变正文内容。

## 页面方案

### 脚本页

- 左侧结构锚点：显示段标题与摘要。
- 左侧策划字段：提供生成/改写脚本所需的结构化输入。
- 中间工作面：展示 Markdown 预览，支持切回原文编辑，不再展示伪分段表单。
- 工作面容器自身承担纵向滚动。

### 分镜页

- 移除页面内联分段函数，直接复用共享 helper。

## 验证标准

- 脚本工作面不再把 `markdown` 或 ````` 代码围栏显示为正文。
- 脚本工作面能直接以预览形式呈现标题、列表、表格等主要结构，表格必须渲染为真实 `<table>`。
- 生成脚本请求必须包含视频主题、目标用户、视频目的、账号定位、拍摄条件、语言要求和禁止内容等策划字段。
- 结构锚点不再出现单独的 `---` 段。
- 分镜页脚本段落数与左侧锚点一致。
- 保存后仍通过原有脚本接口写回完整 Markdown 内容。
