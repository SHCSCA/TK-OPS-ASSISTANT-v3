# M05 基础剪辑工作台 UI 定稿设计

> 对应既有基线：`docs/superpowers/specs/2026-05-13-m05-editing-workspace-assembly-design.md`
> 视觉稿位置：`.superpowers/brainstorm/manual-4460-1778846903/content/editing-workbench-style-v3.html`

## 目标

本设计把 M05 AI 剪辑工作台从“创作链路总装台”推进到“基础剪辑工作台”的视觉基准。它仍然只覆盖基础剪辑，不复制剪映完整功能集；页面只保留素材池、9:16 播放器、基础属性、基础工具和多轨时间线。

本轮设计新增资产中心接入入口，让 M05 能消费当前项目在 M09 资产中心中的真实视频、图片和音频素材，用于替换分镜占位、补充 BGM 或加入时间线。资产中心只作为素材来源，不在 M05 内实现素材库管理、素材市场或复杂文件治理。

## 最终视觉基准

采用第三版样式稿作为实现基准，并在落地时追加三项收敛优化：

- 左侧素材池压缩信息密度：保留缩略图、名称、状态和主操作；格式、分辨率、路径等次级信息进入详情或 hover。
- 时间线采用轨道高度层级：视频轨最高，BGM 和配音轨中等，字幕轨更紧凑。
- 右侧 AI 建议区默认折叠：只显示摘要和数量，展开后才显示具体建议与应用入口。

整体结构：

```text
素材池（分镜 / 配音 / 字幕 / 资产）
┌──────────────────┬──────────────────────────┬──────────────────┐
│ 素材来源          │ 9:16 播放器               │ 基础属性          │
│ 当前项目真实素材  │ 当前片段预览与播放控制     │ 片段 / 素材 / 预检 │
└──────────────────┴──────────────────────────┴──────────────────┘
基础工具：选择 / 移动 / 分割 / 删除 / 磁吸 / 缩放
多轨时间线：视频主轨 / BGM 轨 / 配音轨 / 字幕轨
```

## 素材池设计

左侧素材池包含四个来源 Tab：

- `分镜`：来自 M04 分镜规划，用于镜头结构和无视频时的占位。
- `配音`：来自 M07 配音中心，用于配音轨和段落级音频。
- `字幕`：来自 M08 字幕对齐，用于字幕轨和时间码。
- `资产`：来自 M09 资产中心，用于真实视频、图片、BGM 和可替换素材。

资产 Tab 只显示当前项目可用素材：

- 视频素材：可加入视频轨或替换分镜占位。
- 图片素材：可作为片头、补帧或静态镜头。
- 音频素材：可加入 BGM 轨。
- 需转码素材：可见但不直接入轨，必须先检查或转码。

资产卡片字段：

- 缩略图或波形占位。
- 素材名称。
- 状态：可用、已入轨、需转码、路径缺失。
- 主操作：加入轨道、替换片段、检查。

不在素材卡片中长期展开分辨率、路径、编码和完整元数据，避免素材多时列表过重。素材池窄栏下，资产卡片使用上方“缩略图 + 双行名称/摘要”、下方“状态 + 主操作”的两层结构；来源片段列表必须清除默认列表缩进，不允许出现无内容左侧竖栏；来源片段卡片使用“标题与状态、时间范围、来源轨道”的三层结构。状态和操作不得挤压正文，也不得在 270-330px 素材栏宽度内竖排。

素材池中来自分镜、配音和字幕的来源片段必须可点击，点击后复用时间线片段选择链路，同步播放器、基础属性和时间线高亮。

## 播放器设计

播放器保持 9:16 手机窗口，承担当前片段上下文预览：

- 当前片段已绑定真实视频素材时，显示真实素材预览状态。
- 当前片段仍是分镜占位时，显示中性分镜占位，不伪装成已渲染视频。
- 字幕预览覆盖在手机安全区内，字体不小于 18px。
- 底部控制区包含上一段、播放/暂停、当前时间、进度条和总时长。
- 播放头与底部时间线同步。

播放器的状态文案必须明确来源，例如“资产中心素材，9:16 已匹配”或“分镜占位，待替换真实素材”。

## 时间线设计

时间线是页面核心，不再只显示色块。

轨道层级：

- 视频主轨：最高，显示连续缩略图条。
- BGM 轨：中等高度，显示波形，默认可折叠或空态隐藏。
- 配音轨：中等高度，显示波形和音色/轨道来源。
- 字幕轨：较低高度，显示字幕文本块。

片段规则：

- 同一轨道连续片段默认磁吸贴合，不保留无意义空隙。
- 片段边界通过细线、选中描边和裁剪把手表达。
- 选中片段必须高亮，但不能遮挡文本和缩略图。
- 时间线头部必须显示具体选择摘要，包括片段名或轨道名、状态和时间范围，禁止只显示“已联动到片段”这类泛化文案。
- 播放头使用顶部时间标签 + 竖向红色指示线，穿透所有轨道。
- 时间标尺固定在轨道上方，滚动时保持可读。

基础工具：

- 选择：默认模式。
- 移动：用于片段重新排序或换轨。
- 分割：基于播放头切分。
- 删除：删除选中片段。
- 磁吸：开启后片段首尾自动贴合。
- 缩放：控制时间线密度。

每个工具都必须有明确激活态，避免用户不知道当前处于什么编辑模式。工具栏只表达当前工具状态，具体选中片段摘要由时间线头部和右侧基础属性承担，避免重复展示同一片段名。

## 基础属性设计

右侧属性面板按片段上下文组织：

- 片段信息：名称、轨道、来源、状态。
- 时间参数：开始、结束、时长。
- 素材来源：资产中心路径或创作链路来源。
- 画面 / 音频 / 字幕参数：通过 Tab 切换。
- 预检提醒：只在有问题时突出显示。
- AI 粗剪建议：默认折叠，显示建议数量；展开后展示建议详情和确认入口。

AI 建议不能直接修改时间线。它只能产生可审阅的建议，用户确认后才应用。

## 资产中心边界

M05 只消费资产中心，不管理资产中心。

允许：

- 读取当前项目绑定素材。
- 显示素材状态。
- 将素材加入时间线。
- 用素材替换分镜占位片段。
- 显示素材是否需要转码或是否缺失。

不允许：

- 在 M05 内重建完整资产管理页。
- 做素材市场、模板商城或公共素材库。
- 在 UI 层直接读取文件系统路径。
- 绕过 Runtime 直接拼接资产业务规则。

所有素材读取、转码状态、路径可用性和替换操作都必须通过 Runtime 适配层完成。

## 状态与异常

页面至少覆盖以下状态：

- 加载中：同步时间线、资产和预检状态。
- 空状态：没有时间线或没有资产。
- 正常状态：时间线、素材、播放器和属性面板可用。
- 错误状态：Runtime 请求失败、素材缺失、路径不可用、转码需求未满足。
- 任务状态：资产同步、AI 建议生成、保存、预检、替换片段。

异常提示必须是中文、可恢复、可重试，不暴露 traceback。

## 实现边界

本设计是 UI 定稿，不立即引入复杂剪辑引擎。

进入实现时应拆分模块，避免继续膨胀单文件：

- `WorkspaceAssetRail.vue`：素材来源、资产 Tab、素材卡片。
- `WorkspacePreviewStage.vue`：9:16 播放器、来源状态、播放控制。
- `WorkspaceInspector.vue`：基础属性、预检、AI 建议折叠区。
- `WorkspaceTimeline.vue`：轨道、缩略图、波形、字幕块、播放头。
- `WorkspaceTimelineToolbar.vue`：选择、移动、分割、删除、磁吸、缩放工具。
- `workspaceTimelineViewModel.ts`：轨道排序、片段布局、磁吸展示计算。
- `workspaceAssetViewModel.ts`：资产卡片展示模型和状态映射。

页面仍由 `AIEditingWorkspacePage.vue` 装配，不直接写业务规则。

## 不做事项

- 不做贴纸、特效、滤镜、转场、动画、模板商城和素材市场。
- 不做专业调色、关键帧、遮罩、曲线和复杂混音。
- 不在 UI 层直接读写本地文件。
- 不直接把 AI 建议应用到时间线。
- 不绕过资产中心和 Runtime 适配层。

## 验证要求

实现后至少验证：

- 宽屏下三栏 + 底部时间线铺满内容区，无左右大空白。
- 紧凑窗口下布局折叠但不重叠、不横向撑破。
- 资产 Tab 可显示视频、图片、音频、需转码和缺失状态。
- 时间线片段磁吸贴合、播放头可辨识、轨道高度层级清晰。
- 视频缩略图、音频波形、字幕块在 Light / Dark 下可读。
- AI 建议默认折叠，不抢占基础属性区域。
- 所有 Runtime 错误进入中文错误态并保留重试路径。
