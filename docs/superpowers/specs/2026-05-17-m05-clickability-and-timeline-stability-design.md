# M05 可点击性与时间线稳定性设计

## 目标

本设计收口 M05 AI 剪辑工作台的真实操作体验。当前真实时间线、拖拽、裁剪、资产入轨已经接通，但浏览器复核发现若干“能点但反馈不完整”或“能点但必然失败”的问题。本阶段只解决这些用户正在看的剪辑台问题，不扩展剪辑器大功能。

## 范围

本阶段修复四类问题：

- 素材池来源列表点击后，同步选中时间线片段、播放器上下文和属性面板。
- 时间尺点击使用与轨道片段一致的坐标基准，避免播放头落点偏移。
- 工具栏按当前片段上下文提前禁用无效操作，减少“点击后 Runtime 报错”的常见路径。
- 紧凑窗口下保证播放器区域稳定，尤其是 1280x800 和右侧属性区展开时。

本阶段不做撤销/重做、多选、跨轨拖拽、真实渲染、素材转码入口或 Runtime 契约变更。

## 设计方案

### 素材池来源点击

`WorkspaceAssetRail.vue` 来源列表继续只发出 `select-source-clip` 事件，不直接改 store。页面层 `AIEditingWorkspacePage.vue` 接收事件后调用统一处理函数，完成三件事：

1. 写入 `selectedTrackId` 与 `selectedClipId`。
2. 找到对应片段后把播放头移动到片段起点。
3. 依靠现有 store getter 让播放器、属性面板、素材池 active 状态和时间线选中态一起刷新。

这保持组件职责清晰：素材池负责点击意图，页面负责装配，store 保存全局工作台状态。

### 时间尺坐标

时间线轨道使用 `144px minmax(0, 1fr)` 的两列结构，片段只在 lane 列内布局。时间尺点击如果使用整条 ruler 宽度计算，会把轨道标签列也算进时间区域，导致播放头偏右。

修复方式：`WorkspaceTimeline.vue` 在有轨道 lane 时优先使用第一个 lane 的 `getBoundingClientRect()` 作为时间坐标基准；如果当前没有 lane，则回退到点击元素自身宽度。这样时间尺、空白 lane 和片段拖拽都使用同一条时间轴坐标。

### 工具栏可用性

新增 `workspaceTimelineActions.ts` 保存纯计算逻辑。它输入时间线、选中片段、播放头和步进值，输出：

- `canMoveLeft`
- `canMoveRight`
- `canTrim`
- `canSplit`
- `reason`

连续相邻片段场景下，左移/右移如果会与同轨片段重叠，则提前禁用对应按钮。分割只有播放头严格位于选中片段内部时启用。Runtime 仍保留最终校验，前端只是减少明显无效的用户操作。

### 紧凑布局

桌面宽屏仍保持素材池、播放器、属性面板三栏。容器宽度不足或高度紧张时，布局改成播放器优先：

- 两栏模式：播放器占主列，素材池和属性面板按稳定宽度分列或下沉。
- 单栏模式：播放器、素材池、属性面板垂直排列，时间线保留固定可用高度。
- 播放器列使用 `minmax()` 和 `clamp()` 约束，避免被属性栏压缩成不可用尺寸。

样式只使用现有设计令牌和 CSS variables，Light/Dark 主题继承当前页面变量。

## 状态与异常

- 来源点击没有对应片段时不抛异常，保留当前选中态。
- 时间尺没有可用宽度时忽略点击，避免写入错误播放头。
- 工具栏禁用无效按钮，但 Runtime 错误仍通过现有 store `error` 和页面提示展示。
- 资产 Tab 仍遵守真实资产状态，不新增伪素材或路径判断。

## 测试策略

- 页面测试覆盖素材池来源点击后 active 状态、时间线选中态和属性上下文同步。
- 纯函数测试覆盖相邻片段移动、可分割区间和片段边界场景。
- 页面测试覆盖工具栏禁用状态，保证不能点击必然重叠的左移/右移。
- 浏览器复核覆盖 1280x800 和默认窗口，真实点击 M05 可见按钮和来源项。

## 验收标准

- 素材池“分镜/配音/字幕”列表项点击后有 active 反馈，时间线相同片段同步选中。
- 点击片段上方时间尺后，播放头落在视觉对应时间范围内。
- 连续相邻片段无法移动时，左移/右移按钮提前禁用，不再提交必然重叠的请求。
- 播放头不在片段内部时，分割按钮提前禁用。
- 1280x800 下播放器保持可辨识的 9:16 预览区域。
- `npm --prefix apps/desktop test`、`npm --prefix apps/desktop run build`、`npm run version:check`、`git diff --check` 通过。
