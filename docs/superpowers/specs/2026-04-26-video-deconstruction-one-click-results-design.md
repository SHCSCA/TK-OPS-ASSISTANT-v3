# 视频拆解一键结果页设计

## 目标

视频拆解中心从“导入后自动暴露阶段阻塞”调整为“导入后用户点击开始拆解，一次生成可查看的拆解结果”。转录能力保留为后台可选能力；未配置转录 Provider 时，不阻塞画面理解、基础分段和内容解构。

## 范围

- 视频卡片新增主动作：`开始拆解` / `查看结果`。
- Runtime 新增一键拆解接口：`POST /api/video-deconstruction/videos/{video_id}/deconstruct`。
- 一键拆解按顺序执行：转录尝试、分段、结构提取。
- 转录失败或缺 Provider 只记录 `provider_required` 或 `failed`，不阻塞分段和结构提取。
- 前端结果区采用左侧视频预览、右侧结果 Tab 的样式，Tab 为 `脚本文案`、`视频关键帧`、`内容解构`。
- 继续保留阶段状态与重试入口，但弱化为结果区的辅助状态。

## 非目标

- 不新增真实火山 ASR 调用。
- 不用视觉模型伪造逐字稿。
- 不扩展到剪辑时间线或视频下载。
- 不新增页面，仍在“视频拆解中心”内完成。

## 数据流

1. 用户导入本地视频。
2. Runtime 完成基础媒体解析。
3. 用户点击 `开始拆解`。
4. Runtime 尝试转录；如果缺 Provider，写入可见阶段状态并继续。
5. Runtime 生成基础分段。
6. Runtime 生成结构提取结果。
7. 前端拉取 transcript、segments、structure、stages 并渲染结果 Tab。

## 错误处理

- 文件不存在：一键拆解失败，返回中文错误。
- 转录 Provider 未配置：转录阶段显示“未配置转录 Provider”，结果页脚本文案显示缺项提示。
- 分段或结构提取失败：一键拆解返回失败，保留阶段错误和重试按钮。
- 所有 Runtime 异常必须记录日志，UI 展示中文反馈。

## 验收

- 导入后不会自动开始拆解。
- 点击 `开始拆解` 后，未配置转录 Provider 也能生成分段和结构结果。
- 结果区能切换脚本文案、视频关键帧、内容解构。
- 已完成结构提取后可应用到项目。
- 后端契约、前端交互测试和构建通过。
