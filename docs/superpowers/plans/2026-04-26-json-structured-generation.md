# JSON 真源与组件化展示适配计划

## 背景

脚本生成、脚本改写和分镜生成此前以 Markdown 为主，后续页面再从 Markdown 标题、表格和分隔线中反向解析数据。这导致脚本页、分镜页、配音中心和字幕对齐中心容易把表头、标题和分隔线误当正文，也会在模型输出稍有偏差时出现空白、错列或锚点数量异常。

## 目标

- 新生成结果以严格 JSON 作为唯一真源。
- 前端使用结构化组件渲染出类似 Markdown 文档和表格的视觉效果。
- Markdown 仅保留为旧版本兼容入口，不再作为新链路主格式。
- AI 设置页展示系统默认角色提示词，但用户不可编辑提示词内容。
- 配音中心与字幕对齐中心直接读取结构化口播、字幕和段落字段。

## 范围

- Runtime 脚本版本、分镜版本 DTO 与持久化结构。
- 脚本生成、脚本改写、分镜生成的默认角色提示词。
- JSON 校验、保存、降级错误提示与版本格式标识。
- 脚本页、分镜页、AI 设置页、配音中心、字幕对齐中心。
- 旧 Markdown 数据的兼容展示与复制。

## 非目标

- 不重做项目主链路模型。
- 不改 Provider 配置总线边界。
- 不引入新的业务页面。
- 不把 JSON 直接暴露为用户主要阅读界面。

## 验证

- Runtime 覆盖脚本 JSON 保存、分镜 JSON 保存、AI 能力提示词只读。
- 前端覆盖脚本结构化视图、分镜三种视图、AI 设置保存 payload、配音和字幕下游解析。
- 跑完整前端测试、相关 Runtime 测试和前端构建。
